# Agent 回答质量优化方案

## 优化目标

提升 Agent 对深层次问题的回答质量，解决以下问题：
1. 回复模板过于笼统且篇幅很长
2. 回答不够精准，缺乏深度
3. 缺少多轮对话引导机制

---

## 优化策略

### 1. 知识库增强

#### 导入 GBase8a 官方文档
```bash
# 导入 GBase8a 下载中心文档
coze-coding-ai knowledge add --dataset gbase8a --url "https://www.gbase.cn/download/gbase-8a?category=DOCUMENT"

# 导入 GBase8a 产品手册
coze-coding-ai knowledge add --dataset gbase8a --url "https://www.gbase.cn/docs/gbase-8a/产品手册/doc-introduction"
```

#### 知识库优化
- **数据集名称**：`gbase8a`
- **检索策略**：优先检索 gbase8a 数据集
- **相似度阈值**：0.5（过滤低质量结果）
- **返回数量**：top_k=5

### 2. 回答策略优化

#### 回答优先级
1. **知识库检索**（最高优先级）
   - 所有技术问题必须先从知识库搜索
   - 使用 `search_knowledge_base` 工具，指定 `dataset="gbase8a"`
   - 标注来源："信息来源：GBase8a 官方知识库"

2. **联网搜索**
   - 知识库无结果时，使用 `web_search` 搜索
   - 标注来源："信息来源：联网搜索"

3. **LLM 自身能力**
   - 仅限常识性、非技术性问题
   - 严禁编造技术细节

#### 回答长度控制
- **简单问题**：50-150 字
- **中等问题**：150-300 字
- **复杂问题**：300-500 字
- **文档生成除外**

#### 回答深度要求
- 不仅给出答案，还要解释原理
- 提供可执行的建议
- 标注注意事项和最佳实践

### 3. 多轮对话引导机制

#### 问题不清晰的判断标准
- 缺少关键参数（如表结构、数据量、性能指标）
- 问题过于宽泛（如"如何优化数据库？"）
- 缺少上下文（如"这个 SQL 很慢"）

#### 引导流程

**第 1 轮引导**：
```
为了更准确地回答您的问题，我需要了解以下信息：
- 具体场景/业务背景是什么？
- 涉及的数据规模和表结构？
- 当前遇到的具体问题或目标？
```

**第 2 轮引导**（如仍不清晰）：
```
基于您的描述，我猜测您可能关心以下方面：
1. [可能的需求 1]
2. [可能的需求 2]

请确认您具体想了解哪个方面，或提供更多细节。
```

**第 3 轮（最终引导）**：
```
基于目前的信息，我提供以下建议：

[给出一般性建议]

如果您需要更精准的答案，请提供：
- 具体的业务场景
- 数据结构和规模
- 遇到的问题或目标
```

#### 引导限制
- 最多引导 3 轮
- 每轮只问 2-3 个关键问题
- 避免让用户感到繁琐

## 技术实现

### 1. System Prompt 优化

#### 核心改动
- **精简内容**：删除冗余描述，保留核心信息
- **明确优先级**：强调知识库优先策略
- **长度控制**：限制回答长度为 50-500 字
- **引导机制**：定义多轮对话引导流程
- **深度要求**：要求解释原理和提供可执行建议

#### 关键配置
```json
{
    "config": {
        "max_completion_tokens": 4000  // 降低最大输出长度
    }
}
```

### 2. 知识库检索工具优化

#### 优化点
- 默认参数 `dataset="gbase8a"`
- 相似度阈值 `min_score=0.5`
- 返回数量 `top_k=5`

## 后续优化建议

### 1. 持续改进
- 收集用户反馈，优化回答策略
- 定期更新知识库，添加新的技术文档
- 调整相似度阈值和返回数量

- **问题分类器**：自动识别问题类型（技术/非技术）
- **上下文感知**：基于对话历史动态调整回答策略
- **个性化回答**：根据用户偏好调整语言风格和回答深度

### 3. 监控和分析
- **回答质量监控**：跟踪用户满意度
- **知识库使用分析**：分析知识库检索命中率
